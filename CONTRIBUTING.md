# Contributing to My Brew

Thank you for your interest in contributing to My Brew! We appreciate your help in improving this project. Here are some guidelines to help you get started:

## Reporting Bugs
If you find a bug in the project, please report it using the following steps:
1. **Check for existing issues:** Before creating a new issue, please make sure there isn't already a report for the same problem.
2. **Create a new issue:** Go to the [issues page](https://github.com/Jamie78herndon/my-brew/issues) of the repository and click on the "New issue" button. 
3. **Provide details:** Include the steps to reproduce the bug, your environment (OS, version, etc.), and any relevant screenshots.

## Suggesting Features
We welcome suggestions to improve this project! To propose a new feature:
1. **Open an issue:** Navigate to the [issues page](https://github.com/Jamie78herndon/my-brew/issues) and create a new issue.
2. **Describe the feature:** Provide a clear description of the feature and how it would benefit the project.

## Submitting Pull Requests
If you would like to contribute code:
1. **Fork the repository:** Click on the "Fork" button to create a copy of this repository in your GitHub account.
2. **Create a new branch:** Use a descriptive name for your branch (e.g., `feature/awesome-feature`).
3. **Make your changes:** Implement your changes and ensure they work correctly.
4. **Commit your changes:** Write a clear commit message explaining your changes.
5. **Push to your fork:** Push your changes to your forked repository.
6. **Create a pull request:** Navigate back to the original repository and click on the "New pull request" button. Select your branch and provide an overview of your changes.

## Code of Conduct
Please note that this project adheres to a code of conduct. We expect all contributors to treat each other with respect.

Thank you for contributing!
